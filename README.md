## Формат конфига (.pipe)

Построчный, одна директива на строку:

```
pipeline <имя>      # имя пайплайна
job <имя>           # начать новый job
image <образ>       # docker-образ для текущего job
needs <job>         # этот job ждёт указанный
run <команда>       # шаг (shell-команда)
# ...               # комментарий
```

Пример — `examples/ci.pipe`.

## Структура

- `src/config.*`  — парсер конфига
- `src/runner.*`  — топосортировка job'ов и запуск контейнеров
- `src/main.cpp`  — CLI
- `.github/workflows/build.yml` — CI (собирает проект)
